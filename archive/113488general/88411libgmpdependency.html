Keeley Hoek (Mar 31 2019 at 10:59):

I have not taken the time to try to fully understand the build system, not have I every used CMake for anything, so I can't point exactly to where the distinction between the operating systems is made given default settings, but I did try mac build (on your machine scott) forcing the static option on. However, I get a whole bunch of ""

ld: library not found for -lcrt0.o
clang: error: linker command failed with exit code 1

which looked like a solvable problem (the crt0 has to by lying around somewhere!, I though). But alas, there is this:
https://stackoverflow.com/questions/3801011/ld-library-not-found-for-lcrt0-o-on-osx-10-6-with-gcc-clang-static-flag
Apparently, apple refuses to provide statically linked libSystem.dylib, so you'd have to build it yourself...

This is all to say that you can't just be lazy and pass -static and compile with gcc or clang to fix the problem.
